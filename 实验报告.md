# 工具调用客户端实验报告

## 实验目的

1. 验证基于LLM的工具调用客户端的功能完整性和可靠性
2. 测试AnythingLLM集成功能，验证文档仓库查询能力
3. 评估聊天历史压缩机制的性能和效果
4. 验证输出格式优化的用户体验改进
5. 分析系统在不同场景下的响应速度和稳定性

## 实验环境

- **操作系统**：Windows 10
- **Python版本**：3.6+
- **依赖项**：标准库（os, json, http.client, time, sys, subprocess）
- **配置文件**：`.env`（包含LLM API和AnythingLLM配置）
- **AnythingLLM服务**：运行在 http://localhost:3001

## 实验内容与步骤

### 1. 环境配置验证

**步骤**：
1. 检查 `.env` 文件是否存在并包含必要的配置
2. 验证环境变量加载功能
3. 确认 AnythingLLM 服务是否运行

**预期结果**：
- 系统能够正确读取环境变量
- 能够连接到 AnythingLLM 服务

### 2. 基本工具调用测试

**步骤**：
1. 测试 `list_files` 工具，列出当前目录文件
2. 测试 `create_file` 工具，创建测试文件
3. 测试 `read_file` 工具，读取刚创建的文件
4. 测试 `curl` 工具，访问网络资源

**预期结果**：
- 所有工具能够正常执行
- 工具执行结果正确返回

### 3. AnythingLLM 集成测试

**步骤**：
1. 测试 `anythingllm_query` 工具，查询文档仓库内容
2. 测试特定主题的查询
3. 测试错误处理（API密钥缺失情况）

**预期结果**：
- 能够成功查询文档仓库
- 能够正确处理中文编码
- 错误情况下能够返回明确的错误信息

### 4. 聊天历史压缩测试

**步骤**：
1. 进行多轮对话，触发压缩机制
2. 验证压缩后的对话连贯性
3. 测试压缩过程的性能影响

**预期结果**：
- 当对话轮数超过5轮或上下文长度超过3000字符时，自动触发压缩
- 压缩后的对话保持上下文连贯性
- 压缩过程不会显著影响响应速度

### 5. 输出格式优化测试

**步骤**：
1. 测试工具调用的格式化显示
2. 测试文档仓库查询结果的美化显示
3. 测试统计信息的结构化展示
4. 测试压缩过程的状态提示

**预期结果**：
- 工具调用信息清晰易读
- 文档仓库查询结果直接显示，不再显示原始JSON
- 统计信息结构清晰，包含完整的对话信息
- 压缩过程的状态提示明确易懂

## 功能实现分析

### 1. 核心功能实现

**工具调用机制**：
- 通过 `execute_tool_call` 函数统一处理不同工具的调用
- 支持8种工具：list_files、rename_file、delete_file、create_file、read_file、curl、read_chat_log、anythingllm_query
- 工具执行结果以JSON格式返回，包含状态和数据

**AnythingLLM集成**：
- 使用 `subprocess` 模块调用 curl 命令
- 访问 http://localhost:3001/api/v1/workspace/{workspace_slug}/chat 接口
- 支持中文编码，确保文档内容正确显示
- 从 `.env` 文件读取 API 密钥和工作空间 ID

**聊天历史压缩**：
- 当对话轮数超过5轮或上下文长度超过3000字符时触发
- 采用"保留最近30%，压缩前70%"的策略
- 使用LLM对历史对话进行总结，减少上下文长度

**输出格式优化**：
- 新增 `print_separator`、`print_stats`、`format_tool_call`、`format_tool_result` 函数
- 工具调用和执行结果以标签化格式显示
- 文档仓库查询结果直接显示，不再显示原始JSON
- 统计信息以结构化方式展示

### 2. 技术亮点

1. **模块化设计**：代码结构清晰，功能模块化，易于维护和扩展
2. **错误处理**：完善的错误处理机制，确保系统稳定性
3. **性能优化**：聊天历史压缩机制有效减少上下文长度，提高系统性能
4. **用户体验**：优化的输出格式提升了用户体验，使信息更加清晰易读
5. **安全性**：API密钥从环境变量读取，避免硬编码
