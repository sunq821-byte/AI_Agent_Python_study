# 工具调用客户端（带聊天历史压缩功能 + AnythingLLM集成）

## 项目简介

这是一个基于LLM的工具调用客户端，支持文件操作、网络访问和文档仓库查询，并具有聊天历史压缩功能。当对话轮数超过5轮或上下文长度超过3000字符时，会自动触发历史压缩，提高对话效率和系统性能。

## 功能特性

- **工具调用能力**：支持列出文件、重命名文件、删除文件、创建文件、读取文件、网络访问、读取聊天历史日志和查询文档仓库
- **流式输出**：实时显示LLM的回复内容
- **聊天历史压缩**：当对话超过5轮或上下文长度超过3000字符时，会自动触发历史压缩
- **关键信息提取**：每5轮对话自动提取关键信息，按照5W规则记录到本地日志文件
- **聊天历史搜索**：支持通过特定命令或关键词查找历史聊天记录
- **统计信息**：显示每次对话的耗时、token数和速度
- **AnythingLLM集成**：支持通过API查询文档仓库中的内容
- **美化输出格式**：优化工具调用和执行结果的显示格式，提高可读性

## 环境要求

- Python 3.6+
- 环境变量配置文件 `.env`（位于项目根目录）

## 环境变量配置

在项目根目录创建 `.env` 文件，配置以下内容：

```env
BASE_URL=http://127.0.0.1:1234/v1  # LLM API地址
MODEL=qwen3.5-9b  # 使用的模型
TOKEN=lm-studio  # API令牌

# AnythingLLM配置
ANYTHING_API_KEY=your_anythingllm_api_key  # AnythingLLM API密钥
ANYTHING_WORKSPACE_SLUG=your_workspace_slug  # AnythingLLM工作空间ID
```

## 使用方法

### 交互式模式

```bash
python tool_client.py
```

在交互式模式下，你可以：
- 输入问题或指令与助手交流
- 当对话轮数超过5轮或上下文长度超过3k时，系统会自动压缩聊天历史
- 输入 `exit` 或 `退出` 结束对话

### 命令行模式

```bash
python tool_client.py "你的问题或指令"
```

## 聊天历史压缩机制

- **触发条件**：对话轮数超过5轮或上下文长度超过3000字符
- **压缩策略**：保留最近30%的对话原文，压缩前70%的内容
- **压缩方式**：调用LLM对前70%的对话内容进行总结
- **压缩效果**：减少上下文长度，提高对话效率

## 注意事项

1. 确保 `.env` 文件配置正确，特别是 `BASE_URL` 指向有效的LLM API服务
2. 首次运行时，可能需要安装必要的依赖包
3. 聊天历史压缩会调用LLM，可能会增加一定的响应时间
4. 压缩后的历史以系统提示的形式添加到对话中，确保上下文连贯性

## 故障排除

- **连接错误**：检查 `.env` 文件中的 `BASE_URL` 是否正确
- **权限错误**：确保程序有足够的权限执行文件操作
- **压缩失败**：如果压缩过程中出现错误，系统会使用原始响应作为总结
