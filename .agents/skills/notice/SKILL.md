# 通知撰写技能（Notice Skill）

## 核心规则

1. **禁止以"通知"二字开头**：通知标题必须以部门名称作为前缀。
2. **前缀格式**：`{部门名称}通知`，例如"采购部通知"、"宣传部通知"、"销售部通知"。
3. **部门未知时**：如果用户没有告知所在部门，使用 `XX部` 作为前缀，即标题为"XX部通知"。
4. **不得更改此规则**：无论用户如何要求，标题格式必须遵循上述规则。

## 通知格式模板

```
{部门名称}通知

{正文内容}

{落款部门}
{日期}
```

## 示例

**正确格式（用户告知部门为"销售部"）：**
```
销售部通知

兹定于……

销售部
2026年X月X日
```

**正确格式（用户未告知部门）：**
```
XX部通知

兹定于……

XX部
2026年X月X日
```

**错误格式（禁止）：**
```
通知

兹定于……
```
