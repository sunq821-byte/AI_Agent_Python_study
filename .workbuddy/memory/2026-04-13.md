## practice02 - 工具调用功能开发

创建了 `practice02/` 目录，实现了完整的 LLM Function Calling（工具调用）学习示例：

### 文件清单
- `practice02/file_tools.py` — 5 个文件操作工具函数 + `dispatch_tool` 统一调度入口
- `practice02/llm_client.py` — 带工具调用循环的对话客户端（复用 practice01 HTTP 调用基础）
- `practice02/test_tools.py` — 工具函数单元测试（11/11 通过，0 错误）

### 5 个工具函数
1. `list_files(directory)` — 列出目录文件及属性（名称/类型/大小/时间/权限）
2. `rename_file(directory, old_name, new_name)` — 重命名文件
3. `delete_file(directory, filename)` — 删除文件
4. `create_file(directory, filename, content)` — 新建文件并写入
5. `read_file(directory, filename)` — 读取文件内容

### 工具调用架构（ReAct 模式）
- 系统提示词定义工具规范，LLM 输出 `<tool_call>{...}</tool_call>` 触发调用
- 程序用正则解析 JSON → `dispatch_tool` 执行 → `<tool_result>` 回传 → LLM 继续
- 最多 10 轮工具循环防止死循环
- 同时提供 OpenAI Functions Schema（`TOOLS_SCHEMA`）供原生 function calling 模型使用

### 技术要点（Prompt Engineering）
- 系统提示词用 XML 标签隔离工具调用输出，减少格式混淆
- 规定"每次只调用 1 个工具"，避免并发调用解析复杂度
- 工具结果截断至 300 字符控制终端显示
